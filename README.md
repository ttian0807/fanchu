# 反刍

频繁地记录自己，真的可以改命吗？

如果可以，那该怎么记录呢？
这不是一个 productivity app。不追求高效，不量化你的产出，不会用红色感叹号催你完成任何事。
这不是 to-do list，而是 did list。不告诉你「该做什么」，而是帮你看见「我做了什么」。

反刍 是给这样的人做的：

- 你想建立自己的生活 routine，但总是坚持不下来
- 你有很多同时进行的项目，但它们散落在备忘录、微信收藏和脑子里
- 你是一个启动很慢的人，需要看到自己「确实在动」才能继续动
- 你总是在忙，但回头看不知道自己在忙什么
- 你需要的不是更高的效率，而是学会自我认可
- 你总是在反刍自己的情绪，内耗比输出多
- 你有 ADHD 或类似的注意力困扰，传统的效率工具让你更焦虑而不是更轻松

反刍 把你的 routine、项目进展、杂事和每天的感受放在一条时间轴上。不是为了让你做得更多，而是让你看见——原来我今天做了这些事。

## 功能

- **📋 今日面板** — routine 打卡、项目动态、杂项记录、每日感受，一页看完一天
- **🔄 Routine 管理** — 自定义每日 routine，带输入提示，记录你每天实际做了什么
- **📁 项目 Pipeline** — 自定义项目类型和阶段，跟踪每个项目从开始到结束
- **⏱️ 时间轴** — 所有记录按时间排列，回看任意一天真正发生了什么
- **📤 导入/导出** — JSON 格式，数据完全属于你
- **🕐 自定义时间** — 补记昨天忘了写的事，时间由你定

## 快速开始

```bash
git clone https://github.com/ttian0807/fanchu.git
cd fanchu
npm install
npm run dev
```

打开 `http://localhost:5173`，开始记录。

数据保存在项目根目录的 `tracker-data.json`（自动创建，已 gitignore）。

## 技术栈

- React 19 + Vite
- 纯 CSS + inline styles，无 UI 框架
- 单 JSON 文件存储，零后端依赖
- Vite dev server 中间件提供本地读写 API
